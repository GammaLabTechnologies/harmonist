# Patterns — What Worked, What Didn't

Append an entry when a completed task teaches you something reusable.

- `status: done` — pattern that worked; keep using it.
- `status: blocked` — anti-pattern that cost time; avoid next time.
- `status: rejected` — approach that was tried and discarded; include why.

See `SCHEMA.md` for the entry contract. Use `memory.py append` to add new
entries; direct hand-edits must still pass `memory.py validate`.
